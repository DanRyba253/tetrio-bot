# Бот для [TETR.IO](https://tetr.io/about/desktop/) написанный на Python и Zig

[english version of this readme](README.en.md)

>   [!WARNING]
>   1. Использование ботов в TETRIO карается вечным IP баном без возможности апелляции.  
>   2. Данное ПО во время исполнения, частично берет под контроль клавиатуру пользователя.  
>   А именно:
>       * Клавиши z, x, c
>       * Пробел
>       * Клавиши левой и правой стрелок
>       * Клавишу активации, выбранную при запуске
>
>   Автор снимает с себя ответственность за любые последствия использования данного ПО, включая, но не ограничиваясь санкциями со стороны разработчиков TETRIO, проблемами с вашим компьютером и испорченными файлами.  
>   Данное ПО предоставляется "как есть", без каких-либо гарантий.   Используйте на свой страх и риск.

## Поддерживаемые системы
* Windows x64
* Linux x86_64

## Инструкция пользователя

### Windows

Перед использование убедитесь что на вашей системе установлен Python версии не менее 3.11

1. Скачайте репозиторий и зайдите в папку с файлом `start.bat`
2. Запустите файл `start.bat`
>   [!NOTE]
>   Первый запуск может занять несколько минут пока устанавливаются необходимые библиотеки.  
>   Перед первым запуском убедитесь в наличии подключения к интернету.
3. В открывшемся графическом интерфейсе выберите адаптер `Windows x64`
4. Настройте бота по желанию. Подробнее о доступных настройках см. [соответсвующую секцию](#доступные-настройки-бота).
5. Запустите бота нажатием на кнопку `Запустить`
6. Перейдите в полноэкранное окно TETRIO
7. В Config поставьте следующие настройки:
    * Gameplay:
        * Board bounciness - 0%
        * Grey out locked hold piece - off
        * Action text - off
    * Video & Interface:
        * Particle count - few
        * Background visibility - 0%
8. Начните игру в режиме Zen или Custom
9. Для активации/деактивации бота, нажмите на клавишу активации, выбранную в графическом интерфейса

### Linux (X11)

На данный момент, нет адаптера под X11. Если есть желание и возможность его написать, см. [соответствующую инструкцию](#инструкция-по-созданию-своего-адаптера)

### Linux (Wayland)

Так как Wayland не поддерживает общих методов регистрации глобальных горячих клавиш, создания скриншотов и симуляции нажатия на клавиш клавиатуры, приходится писать адаптеры специально для каждой среды рабочего стола и каждого оконного менеджера.

На данный момент, есть адаптер только для оконного менеджера Niri.  
Если есть желание и возможность написать адаптер для своей системы,
см. [соответствующую инструкцию](#инструкция-по-созданию-своего-адаптера).

### Linux (Niri)

1. Убедитеть, что установлены следующие пакеты или их эквиваленты для вашего дистрибутива:
    * `grim`
    * `ydotool`
    * `tk`

2. Создайте пустой файл `~/.config/niri/tetrio-binds.kdl` и импортируйте его в ваш конфиг для Niri.

3. Запустите сервис `ydotoold`:  
```sh
sudo ydotoold --socket-perm=666 --socket-path=/run/user/1000/.ydotool_socket --mouse-off
```
3. Следуйте инструкции для Windows со следующими правками:
    * Вместо `start.bat` используйте `start.sh`
    * Выберите адаптер `Linux x86_64 (Niri)`

## Доступные настройки бота

1. `Клавиша активации` - клавиша, с помощью которой пользователь может активировать/деактивировать бота во время игры
2. `Задержка после хода` - время, которое бот ждет после совершения хода в миллисекундах. Если бот просто кладет фигуры вниз, попробуйте слегка увеличить этот параметр.
3. `Попыток сделать скриншот` - количество попыток сделать очередной скриншот и запарсить его, которые бот предпримет, перед тем как совершить вынужденный ход (нажатие на пробел).
4. `Задержка после вынужденного хода` - время, которое бот ждет после совершения вынужденного хода.
5. `Искать углов`  количество углов, которые парсер скриншотов пытается найти.
6. `Расстояние между углами` - минимальное расстояние между углами найденными парсером.

## Инструкция по созданию своего адаптера

1. Скопируйте `python-src/adapters/adapter-template.py` в новый файл
2. Реализуйте все методы способом наиболее подходящим для вас
3. В файле `gui.py` добавте опцию выбора вашего адаптера
    ```python
    self.adapter_options = ["Windows x64", "Linux x86_64 (Niri)", "Мой крутой адаптер"]
    ```
4.  В файле `bot.py` добавьте код обработки вашего адаптера
    ```python
    if conf.adapter == "Windows x64":
        from adapters import windows
        adapter_module = windows
        engine_path = "../engine/engine-windows-x86_64.exe"

    if conf.adapter == "Linux x86_64 (Niri)":
        from adapters import niri
        adapter_module = niri
        engine_path = "../engine/engine-linux-x86-64"

    # Ваш код
    if conf.adapter == "Мой крутой адаптер":
        from adapters import my_adapter
        adapter_module = my_adapter
        engine_path = "../engine/engine-linux-x86-64"
    ```
